### [BodySmart](https://www.bodysmartapp.com/)

![Screenshot of BodySmart](/assets/img/bodysmart.png)
- A fitness planning, tracking, and coaching progressive web app using React, Python, and PostgreSQL.
- The app provides a workout plan that is customized to the user’s available gym equipment and workout duration.
- Users can then “start” their workout through the app, which trains them on each exercise while a custom timer
counts down. 
- At the end of the workout, users can take pictures (image uploading via Amazon S3) to track their
progress.
- Propietary database of more than 2,000 activities, each with their own page for SEO.

### GoGaiGai Personal Safety App 

![Screenshot of GoGaiGai](/assets/img/gogaigai.jpg)
- A group project that won “Best App” and “Best Pitch” out of four entries to NEXT Academy’s bi-monthly pitch
night.
- A safety companion progressive web app aimed for solo travelers built using Python, React, and PostgreSQL.
- Features included a user-generated crime map, ability to add an itinerary check-in, notification of emergency
contacts if user is unresponsive to alerts, and mock phone/video calls that allowed the user to discreetly analyze
their surroundings.
- Go GaiGai means “to go take a walk” in Cantonese, reflecting its use while exploring a new city.
- Watch our interview on [Youtube](https://www.youtube.com/watch?v=eDxSzNyTwog) or [read about it](https://www.nextacademy.com/blog/go-gai-gai-personal-safety-app-for-female-traveler).

### [Instagood](https://nextagram-react.netlify.app/)

![Screenshot of Instagood](/assets/img/instagood.png)
- Side project creating Instagram with a twist using Python and Flask.
- Features include sign-up/login (password encryption, authentication, authorization, cookies, sessions, OAuth),
image uploading using Amazon S3, sending emails using Sendgrid API, handling background jobs with Redis/RQ,
collecting payments via Braintree, and creating public RESTFUL API endpoints.
- Challenges included building complex model relationships (e.g. self-referential many-to-many) using PostgreSQL.
- See code on [Github](https://github.com/kharissa/instagood)
